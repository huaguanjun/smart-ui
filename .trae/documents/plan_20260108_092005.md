## 修复 Ant Design Vue 表格数据显示问题

### 问题分析
1. 已修复数据属性名问题：将 `data` 映射到 `dataSource`
2. 缺少 `rowKey` 属性：Ant Design Vue 表格需要唯一标识每一行数据
3. 数据包含 `id` 字段：可以作为默认的 `rowKey`

### 修复步骤

1. **修改 Ant Design Vue 表格适配器**
   - 在 `smart-form-core/src/adapters/ant/Table.vue` 中添加 `rowKey` 属性处理
   - 如果提供了 `rowKey`，则使用提供的值
   - 如果没有提供 `rowKey`，默认使用 `id` 字段作为 `rowKey`

2. **重新构建核心库**
   - 运行 `npm run build` 重新构建核心库

3. **验证修复效果**
   - 检查 demo 项目是否成功加载修复后的核心库
   - 查看预览页面，确认 Ant Design Vue 表格显示数据

### 预期结果
- Ant Design Vue 表格能够正常显示数据
- 无需手动配置 `rowKey`，自动使用 `id` 字段
- 支持自定义 `rowKey` 属性
- 与 Element UI 表格显示效果一致
