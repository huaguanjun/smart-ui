# 为 Smart Form 组件添加字段级插槽支持

## 问题分析
当前 Smart Form 组件支持通过全局插槽添加自定义表单内容，但不支持针对特定字段使用插槽进行特殊处理。用户希望能够对表单中的某一项进行自定义渲染。

## 解决方案
为每个自动生成的表单字段添加命名插槽支持，允许用户针对特定字段进行自定义处理。

## 实施步骤

### 1. 修改 Ant Design Vue Form 适配器
- 文件：`/Users/apple/Desktop/migu/test/middleUI/smart-ui/src/adapters/ant/Form.vue`
- 为每个表单字段添加命名插槽：`<slot :name="field.name" v-bind="{ field, model: props.model }"></slot>`
- 在默认控件渲染前检查是否有插槽内容

### 2. 修改 Element Plus Form 适配器
- 文件：`/Users/apple/Desktop/migu/test/middleUI/smart-ui/src/adapters/element/Form.vue`
- 同样为每个表单字段添加命名插槽
- 保持与 Ant Design Vue 适配器的一致性

### 3. 优化字段配置类型
- 文件：`/Users/apple/Desktop/migu/test/middleUI/smart-ui/src/core/types.ts`
- 确保 `FieldConfig` 类型支持插槽相关配置

### 4. 更新 SmartForm 组件
- 确保主组件正确传递插槽内容给适配器

## 预期效果

1. 用户可以通过命名插槽自定义特定字段的渲染：
   ```vue
   <smart-form :model="formData" :fields="fields">
     <template #username="{ field, model }">
       <!-- 自定义用户名字段渲染 -->
       <custom-input v-model="model.username" placeholder="自定义用户名输入"></custom-input>
     </template>
   </smart-form>
   ```

2. 保持向后兼容性，未提供插槽内容的字段仍使用默认渲染

3. 支持两种 UI 库（Element Plus 和 Ant Design Vue）

## 技术细节

- 使用动态命名插槽：`#${field.name}`
- 传递字段配置和模型给插槽：`v-bind="{ field, model }"`
- 条件渲染：先检查插槽是否存在，不存在则使用默认控件
- 保持代码结构清晰，便于维护和扩展
