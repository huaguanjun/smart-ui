# 修复 Ant Table 适配器无法获取 props 数据的问题

## 问题分析
1. **现象**：Ant Table 适配器无法获取 `data` 属性，导致 `dataSource` 映射为 undefined，表格渲染不出来
2. **根本原因**：在 Vue 3 的 `<script setup>` 中，使用 `defineProps<{ columns?: any[]; [key: string]: any }>()` 定义 props 时，Vue 会根据接口的明确属性来生成 props 定义，而忽略索引签名
3. **影响范围**：所有通过索引签名传递的 props（包括 `data`）都无法被正确接收

## 解决方案
修改 Ant Table 和 Element Table 适配器的 props 定义方式，确保它们能够正确接收所有传递过来的 props。

## 实施步骤
1. **修改 Ant Table 适配器**：更新 `/Users/apple/Desktop/migu/test/middleUI/smart-ui/src/adapters/ant/Table.vue` 中的 props 定义
2. **修改 Element Table 适配器**：更新 `/Users/apple/Desktop/migu/test/middleUI/smart-ui/src/adapters/element/Table.vue` 中的 props 定义
3. **验证修复效果**：重新构建并运行项目，确保表格能够正常渲染

## 预期效果
- 适配器组件能够正确接收所有传递过来的 props
- `data` 属性能够被正确获取，`dataSource` 映射正常
- 表格能够正常渲染数据
