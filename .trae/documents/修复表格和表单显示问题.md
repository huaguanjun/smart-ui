## 修复表格和表单显示问题

### 问题分析
1. **组件注册失败**：使用 `<script setup lang="ts">` 语法的组件默认没有 `name` 属性，导致插件注册时无法获取组件名称
2. **动态组件属性传递**：需要确保动态组件正确接收所有必要的属性
3. **适配器组件实现**：需要检查表单和表格适配器组件是否正确实现

### 修复步骤

1. **为组件添加 name 属性**
   - 在 `SmartForm.vue` 中添加 `<script>` 标签，设置组件名称
   - 在 `SmartTable.vue` 中添加 `<script>` 标签，设置组件名称

2. **优化组件注册逻辑**
   - 修改 `index.ts` 中的组件注册方式，使用固定名称注册组件

3. **检查表单适配器组件**
   - 查看 `ElementForm` 和 `AntForm` 组件的实现
   - 确保它们正确接收和处理表单属性

4. **验证修复效果**
   - 运行 demo 项目，检查表单和表格是否正常显示
   - 测试不同 UI 库的切换效果

### 预期结果
- 表单和表格组件能够正常显示
- 支持通过 JS 配置自动生成表单控件和表格列
- 能够切换不同的 UI 库
- 实现"少操作模板多操作 JS"的需求
